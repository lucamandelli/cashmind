Color System

The complete color foundation for CashMind, derived from the logo. Every color is a named token with a light and dark value — built so the same component looks right in both themes without you touching it twice. Toggle the theme above; the whole page (and every example) responds live.

01

Brand colors

Three fixed brand colors carried straight from the logo. Emerald is the identity and the one action color — used sparingly so it stays meaningful. Bright emerald is its dark-mode counterpart (more luminous, so it holds up on near-black). Ink is the darkest neutral and the base of the dark theme.

02

Emerald scale

A full tint-and-shade ramp around the brand emerald. You rarely pick from this directly — the theme roles below do — but it's here so hovers, pressed states, subtle backgrounds, and borders all stay in the same color family. 500 is the brand anchor.

03

Neutral scale

A slightly cool gray ramp (tuned to the ink) that does the quiet, heavy lifting: page and card backgrounds, text, dividers, and borders. A serious finance app is mostly neutral — color is the exception, not the rule.

04

Semantic colors

Meaning-carrying colors. In finance these are not decoration — they communicate money movement and status, so they're consistent everywhere. Note positive reuses the brand emerald (gains feel on-brand), while negative is the one place red appears. Each has a solid color and a soft "subtle" background tint.

05

Theme roles

The tokens you actually build with. Instead of picking raw hex, you use a role — --surface, --text-2, --border — and it resolves to the right value for the current theme. Values shown update live as you toggle.
